> Consider the following two 7-bit floating-point representations based on the

> 1. Format A
>   - There are k = 3 exponent bits. The exponent bias is 3.
>   - There are n = 4 fraction bits.
> 2. Format B
>   - There are k = 4 exponent bits. The exponent bias is 7.
>   - There are n = 3 fraction bits.
> Below, you are given some bit patterns in format A, and your task is to

|     Bits |    Value |     Bits |    Value |
|----------|----------|----------|----------|
| 011 0000 |        1 | 0111 000 |        1 |

| 010 1001 |    25/32 | 0110 100 |      3/4 |
| 110 1111 |      124 | 1110 000 |      128 |

Format A (left), Format B (right)

1. 101 1110

Format A:
```
Bits = 101 1110

e = 5
E = e - bias = 2
f = .1110 = 7/8
M = 1+f = 15/8

Value = 15/8 * 2^2 = 14/4 = 7/2
```

Format B:
```
E = 2
e = E + bias = 11 = 1011

f = .111

Bits = 1011 111
Value = 7/2
```

2. 010 1001

Format A:
```
Bits = 010 1001

e = 2
E = e - bias = -1
f = .1001 = 9/16
M = 1+f = 25/16

Value = 25/16 * 2^-1 = 25/32
```

Format B:
```
e = E + bias = 6 = 0110
f = round(.1001, 3) -> .100 = 1/2

Bits = 0110 100
Value = 3/2 * 2^-1 = 3/4 = 24/32
```

3. 110 1111

Format A:
```
Bits = 110 1111

e = 6
E = e - bias = 4
f = .1111 = 15/16
M = 1+f = 31/16

Value = 31/16 * 2^6 = 124
```

Format B:
```
round(1.1111, 3) -> 10.000 (move point once to the left, incrementing e by 1)
f = .000
e = E + bias + 1 = 14 = 1110

Bits = 1110 000
Value = 2^7 = 128
```

4. 000 0001
